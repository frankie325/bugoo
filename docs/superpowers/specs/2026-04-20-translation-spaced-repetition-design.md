## 4. 系统架构

### 4.1 技术方案：纯本地极简

```
DeepL API（外部）
      ↑ HTTPS
Tauri 应用（Rust + React）
  ├── 划词捕获层（Rust：全局热键/选区监听）
  ├── 翻译浮窗（React 组件）
  ├── 艾宾浩斯引擎（Rust：SM-2 算法）
  ├── 通知调度器（Rust：系统通知 API）
  └── SQLite 本地存储
      ↓（可选）
  iCloud / Dropbox 文件同步
```

### 4.2 为什么这样设计？

- **DeepL 而非本地词库**：多语言支持需求，本地词库体积大，DeepL 免费额度足够
- **艾宾浩斯放 Rust 层**：后台调度任务比 JS 更稳定，前端只负责 UI
- **TTS 用系统级 API**：macOS `NSSpeechSynthesizer` / Windows SAPI，无须第三方音频服务
- **不做账户系统**：云同步委托给系统级 iCloud/Dropbox，无须维护服务端
- **纯本地运行**：离线也能工作，隐私最好

## 6. 艾宾浩斯算法（SM-2 简化版）

```
初始值：ease_factor=2.5, interval=1, repetitions=0

点"认识"时：
  repetitions += 1
  IF repetitions == 1: interval = 1
  IF repetitions == 2: interval = 3
  IF repetitions >= 3: interval = ROUND(interval * ease_factor)
  ease_factor = ease_factor + 0.1（最大 2.5）
  next_review_at = NOW + interval 天

点"不认识"时：
  repetitions = 0
  interval = 1
  ease_factor = MAX(1.3, ease_factor - 0.2)
  next_review_at = NOW + 1 天
```

**温和提醒策略：** 每天最多触发 1-2 次通知，时间点可选（上午/下午/晚上各一个时间窗口）。

## 7. UI/UX 设计方向

### 7.1 四个核心视图

1. **划词翻译浮窗** — 单词 + 翻译 + 发音按钮 + "添加到生词本"按钮，浮于所有窗口之上
2. **生词本列表** — 单词 + 释义 + 标签 + 复习状态，支持搜索和按标签筛选
3. **单词详情卡** — 点击生词查看完整信息：翻译、标签、笔记、发音、复习历史
4. **复习通知卡片** — 系统通知横幅内直接显示"认识/不认识"按钮

### 7.2 视觉风格

- 双主题自适应（跟随系统浅色/深色模式）
- 柔和浅色：米白/浅灰背景 + 低饱和度蓝绿点缀
- 深色沉浸：墨色背景 + 暖白文字
- 核心原则：不抢注意力，让用户专注在单词和翻译上

### 7.3 设计约束（保持简洁）

- 生词本列表默认只显示单词+释义，标签默认折叠显示，不干扰核心阅读体验
- 笔记仅在点击展开单词详情时才可见，不在列表中喧宾夺主
- 发音按钮默认隐藏，用户主动点击才发声，不自动播放打扰他人
- 没有复杂的文件夹层级，只支持扁平的标签系统
- 只有一个复习队列：添加生词 → 自动进入复习 → 无需用户选择"学哪个"

## 9. 技术风险与缓解

| 风险 | 缓解方案 |
|------|----------|
| DeepL API 限流 | 实现本地缓存 + 请求合并 |
| 划词在某些 App 不工作 | Tauri 提供 accessibility API，主流 App 兼容 |
| 多设备冲突 | iCloud/Dropbox 自带冲突解决，我们只读写文件 |
| 通知被系统屏蔽 | 首次启动引导用户授权通知权限 |
| TTS 多语言支持有限 | 系统 TTS 优先使用，失败时静默降级（不影响核心复习功能） |
